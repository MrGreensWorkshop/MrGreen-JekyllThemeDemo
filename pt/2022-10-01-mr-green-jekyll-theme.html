Introdução

Mr. Green é um tema para vários idiomas criado com Jekyll e totalmente compatível com GitHub Pages.

Eu estava criando meu wbesite e pensei em fazê-lo como um modelo que eu poderia compartilhar com a comunidade do código aberto. Esse é o motivo que me levou a criar este tema. Eu trabalhei muito nesse projeto e eu ficaria muito feliz caso você decida me ajudar. Obrigado.

Recursos disponíveis

  • Website para vários idiomas.
    • Inglês (padrão), Japonês, Português do Brasil, Francês
  • Recomendação de idioma
  • Auto navegação com possibilidade de habilitar e desabilitar os ícones.
  • Layout para Principal, Sobre, Arquivos, Lista-de-Posts, Links, Projetos e mais
  • Possibilidade de trocar as cores (claro e escuro)
  • Possibilidade de adicionar Auto Contraste
  • Auto referência a imagens (não há a necesisdade de adicionar o caminho completo para imagens. Use somente : antes do nome do arquivo.)
  • Visualizador de imagens fácil e visualizador de imagens
  • Rodapé com diferentes licenças para uso
  • Tabela móvel de conteúdo para posts
  • Opção de compartilhar posts.
  • Os posts podem ser listados por categoria ou marcadores.
  • Comentários em posts
    • Giscus
    • Disqus
  • Seleção de diferentes paginações e modos de carregar conteúdo
  • Mapa do site
  • Search Engine Optimization (SEO)
    • Schema Markup
    • Open Graph
    • Twitter
  • Analytics (Google Analytics)
  • Consentimento para usar cookies
  • Formulário de contato (Formulários do Google)
  • Buscador no site
  • Compressão de código (HTML JS SCSS)
  • Suporte a dispositivos móveis
  • Compatível com dispositivos móveis (testado com Android e IOS)
  • Estrutura de pastas bem organizado para desenvolvedores (testado no Chrome, Safari e Firefox)

Instalação

GitHub pages

  1. Faça um fork do repositório.

  2. Edite _config.yml e o link do url e suba suas alterações.

    url: "https://seu_nome_de_usuario.github.io"
  3. Renomeie o repositório para seu_nome_de_usuario.github.io
  4. Veja o estatus da implementação em Actions no repositório.
  5. Quando este ficar verde, seu site estará online e rodando em https://seu_nome_de_usuario.github.io

Rodando localmente

  1. Instale o Jekyll para seu sistema operacional
  2. Clone ou faça o download do repositório, no prompt de comando vá para a pasta e rode o comando bundle install
  3. Build using bundle exec jekyll serve --watch --host 0.0.0.0 --safe command
    • com o atributo --host 0.0.0.0 você consegue acessar um servidor web que está na sua rede.
    • com o argumento --safe você pode ter certeza que nenhum plugin de terceiros será adicionado. (para desenvolvimento voltado ao Github)
  4. Sua página estará rodando no endereço 0.0.0.0:4000/.

Créditos

Eu quero agradecer esses projetos que proporcionaram a oportunidade de construir meu wbe site.

  • Jekyll é um gerador de site estático. É capaz de usar textos escritos na sua linguagem favorita usando markup e usar layouts para criar web sites estáticos. Você consegue altear todo o site, URLs, as informações mostradas na página e mais. É um incrível projeto que é mantido por voluntários.

  • GitHub Pages hospedado diretamente no seu repositório do GitHub. Suba suas alterações e seu site será gerado automaticamente.
